## 🧠 How I Build

I prefer practical architecture over shiny architecture.

My usual approach:

1. Start with the business problem
2. Shape the data correctly
3. Build the smallest working system
4. Make it modular
5. Automate the repetitive parts
6. Keep the door open for scale without pretending every project is Netflix

I’m comfortable working close to the metal when needed: plain PHP, vanilla JavaScript, MySQL, custom APIs, and shared hosting constraints. I’m also comfortable planning forward into Node, TypeScript, Python workers, PostgreSQL, Redis queues, Docker, and VPS-based infrastructure when the project earns that complexity.
